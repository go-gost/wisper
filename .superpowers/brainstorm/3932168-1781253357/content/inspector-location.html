Inspector data — where should it live?

How should users access traffic inspection from a tunnel/entrypoint detail page?

A

Inline Section

A collapsible "Traffic Inspection" panel below the stats grid on the existing detail page. Toggle to expand/collapse inspector data within the same page — no navigation away.

B

Separate Tab

A "Traffic" tab button at the top of the detail page that reveals a full inspector interface. The info card + stats grid remain above, with a tab bar to switch between "Overview" and "Traffic" views.

C

Dedicated Route

A "View Traffic" button on the detail page navigates to a separate full-page route like /tunnel/:type/:id/traffic. Inspector gets its own deep page with protocol tabs, query history, and live streaming — full-screen, no tunnel context clutter.

D

Slide-out Drawer

A button opens a slide-out panel from the right edge, overlaying the detail page. Shows inspector data while keeping the tunnel info visible behind a semi-transparent backdrop.
